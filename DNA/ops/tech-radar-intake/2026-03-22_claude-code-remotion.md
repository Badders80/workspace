## Overview

This stood out because it reframes video editing as code generation plus prompt
orchestration. The reel showed Claude Code driving a Remotion workflow for
split-screen edits, subtitles, cuts, and transitions, which suggests a much
faster path for demos or marketing videos than a fully manual edit flow.

It feels especially relevant because it sits beside existing design and motion
experiments instead of replacing them. Remotion brings a programmable video
surface, while Claude Code becomes the editing copilot that can scaffold or
revise the video logic quickly.

## How It Works

The reported workflow starts with a prompt describing the desired video edit.
Claude Code then generates or modifies the React-based Remotion components that
define layout, timing, subtitles, transitions, and rendering. The linked Notion
guide was source-limited in this pass, so the extracted understanding comes
mainly from the reel plus the known Remotion model.

- Prompt describes the edit goals.
- Claude Code shapes the Remotion components.
- Remotion renders the final video output from code.

## Takeaways

- Strong example of prompt-driven editing for code-native video workflows.
- Could complement Magic Animator rather than duplicate it.
- Makes more sense as a bounded experiment than as a theory-only note.

## Risks / Caveats

- Remotion still requires code-level debugging when things go wrong.
- Render time and project complexity can grow quickly.
- The setup guide was not fully extractable in this pass, so implementation
  details remain partly source-limited.

## If I Revisit This

1. Recreate the workflow with a simple split-screen and subtitle example.
2. Compare the speed and quality against the current Magic Animator path.
3. Promote it only if the code-driven video flow proves faster on a real task.
