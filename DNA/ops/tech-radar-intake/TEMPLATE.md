# Tech Radar Note Template

Use this for the durable repository note.

Important: this file is for the reading-library entry, not the final
fit-for-us verdict. Keep `ADOPT`, `TRIAL`, `ASSESS`, or `ARCHIVE` in the
separate radar wizard output and in `TECH_RADAR.md`.

Processor output order:
1. Title
2. Source
3. Human Review
4. Technical / Fit Review
5. Tech Radar Recommendation

This template covers the durable repository-note part of that flow.

```md
# {Topic}

Tool:
Category:
Reviewed:
Source:
Source confidence: High | Medium | Low
Links:
- ...

## Overview

Write 1-2 short paragraphs.
Combine what caught your eye, what it is, and your human review into a single
natural mini brief.

## How It Works

Start with a short paragraph.
Add bullets only if the mechanism benefits from steps.

## Takeaways

- What stands out
- What seems promising
- What to keep in mind later

## Risks / Caveats

- Maturity
- Cost
- Lock-in
- Source-quality caveats
- Privacy or operational concerns

## If I Revisit This

1. What I should read or test next
2. What question needs answering
3. What would make this more or less relevant
```
