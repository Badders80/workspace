## Overview

This stood out because it tackles a real pain point directly: AI prose that
sounds obviously AI-generated even when the substance is good. The repo frames
that problem as a reusable skill plus scoring system, which makes it feel like
an immediate companion to the existing prompt and evaluation work rather than a
new platform to manage.

The strongest fit is as a finishing or quality-gate layer for Claude outputs
that need to read cleanly for investors, breeding notes, or internal writing.
It looks like a practical, low-effort upgrade as long as it is used carefully
enough to avoid turning every output into the same flattened voice.

## How It Works

The repository provides a markdown skill and supporting reference material that
can be loaded into Claude-facing workflows as instructions, rewrite guidance,
or a scoring rubric.

- It identifies common AI phrases and structural tells to avoid.
- It pushes the model toward direct, plain, human-sounding prose.
- It can also be used as a scoring pass to judge whether an output still feels
  synthetic.

## Takeaways

- Strong fit as a companion to the current prompt evaluation work.
- Useful for writing-heavy outputs where tone matters as much as correctness.
- The scoring angle makes it more practical than a vague "write better"
  reminder.

## Risks / Caveats

- Overuse could make outputs feel rigid or over-edited.
- A single anti-slop style can become its own tell if applied too hard.
- It needs real-world testing on the team's own writing surfaces before it
  earns a bigger role.

## If I Revisit This

1. Load the skill and reference files into the main Claude workflow and test it
   on a recent memo.
2. Compare the scored output against a normal pass to see whether clarity
   improves without flattening voice.
3. Keep only the parts that genuinely help and strip out anything that feels
   too robotic.
