# Marketplace Glossary

Updated: 2026-04-07  
Status: active Phase 0 vocabulary surface

## Purpose

Canonical Phase 0 glossary and language guardrails for the Evolution Stables
domestic marketplace workstream.

Use this file for investor-facing and operator-facing wording in current Phase
0 artifacts only.

## Core Terms (Phase 0 Only)

- `Ownership Stake` - The sanctioned term for a fractional investor holding in
  a horse participation opportunity.
- `Investor Journey` - The end-to-end investor path from discovery through
  onboarding, stake selection, and portfolio visibility.
- `Discovery` - The stage where a prospective investor first finds and
  understands available Ownership Stakes.
- `Onboarding` - The initial investor setup flow, including account setup and
  identity checks where required.
- `KYC` - Know-your-customer verification process, handled through a
  provider-agnostic integration layer.
- `Provider-Agnostic` - Designed so external provider choices can change
  without rewriting core marketplace contracts.
- `Ticket Contract` - The required execution frame for each task: goal,
  allowlist, budget rule, expected artifact, definition of done, owner and
  approver, runtime route, and result log.
- `Allowlist` - The approved write-path boundary for a ticket. No writes are
  allowed outside it.
- `Definition of Done` - Explicit completion criteria that must be satisfied
  before verification and approval.
- `Verification Review` - Independent check against ticket requirements and
  quality criteria before final approval.
- `Pass/Hold/Fail` - Standard review outcomes:
  - `Pass`: ready for approval
  - `Hold`: gaps must be corrected
  - `Fail`: output misses core requirements and needs rework
- `Blocked` - A task state where progress cannot continue due to missing input,
  conflict, or constraint.
- `Requeue` - Return of a task to the queue after revision or clarified scope.
- `Runtime Route` - The selected execution path for a ticket (for Phase 0,
  approved free-route execution only).
- `Zero-Spend Rule` - The active trial budget cap of `0`; only free routes are
  permitted unless the Founder explicitly changes the cap.

## Banned Language (Phase 0)

Do not use the following in investor-facing copy:

- `shares`
- `units`
- `guaranteed returns`
- `risk-free`
- `safe bet`
- `financial advice`
- `wealth product`
- `investment certainty`
- `capital guaranteed`
- any wording that implies advice, certainty, or regulated claims not approved

## Investor-Facing Language Rules (Phase 0)

- Use British English spelling and punctuation.
- Use `Ownership Stake` consistently; do not substitute `shares` or `units`.
- Keep wording plain, specific, and operational.
- Avoid speculative future-state claims.
- Avoid legal, compliance, or performance claims beyond approved wording.
- Do not present guidance as financial advice.
- Keep trust language factual and process-based, not promotional hype.

## Scope Boundary

This glossary is intentionally limited to Phase 0 operating needs.  
Additions require a ticketed update and review.
